## 3. 策略评估

### 双方一致同意的

- **策略有微弱 edge**，通过了交叉验证
- **本质不是"跟随操纵者"**，而是均值回归 + 流动性溢价
- **薄盘口市场**中价格更容易偏离也更容易回归
- **不加仓（MAX=1）** 优于加仓，所有扫描一致
- YES/NO **参数必须分离**

### 双方分歧

| 话题 | 我们的发现 | Codex 看法 |
|------|-----------|-----------|
| 参数稳定性 | 3 轮回测一致，已锁定 | 213 窗口不够，需分段验证 |
| TP 设计 | 0.85/0.88 不对称最优 | 建议分批止盈（0.70 出一半） |
| 利润放大 | Size 线性放大即可 | 先加交易成本验证 |
| 数据量 | 2 天够初步结论 | 需 1,000+ 窗口确认 |

## 4. 参数优化建议

### 共识：当前参数可以保留不变

| 参数 | 双方都同意 |
|------|-----------|
| MAX_ENTRIES=1 | ✅ |
| YES_HIGH=0.45 | ✅ |
| SL 不对称 | ✅ |
| YES/NO 分离 | ✅ |

### Codex 额外建议（我们没试的）

**分批止盈**
```
TP1: 0.70 卖一半，TP2: 0.85 卖剩余
```
理由：很多交易可能在 0.70-0.85 之间波动，分批能先锁一半利润

**时间止盈**
```
最后 60-90 秒如果价格超过 0.70 就退出
```
理由：临近结算价格波动大，提前走人

**动态止损**
```
价格从最高点回撤一定比例退出（trailing stop）
```
理由：固定止损可能被短期波动触发

**最后 60 秒禁止入场**
比当前的 15 秒保护更严格

**独立冷却时间**
YES/NO 独立冷却，而不是共享

### 我们已证明但 Codex 没看到的

- TP 0.85/0.88 是最优不对称值（我们跑了 1,225 种组合确认的）
- 4% 的 TP 是假跳（960 验证）
- MAX_ENTRIES=1 在 9,216 种组合中确认最优
- YES_LOW=0.25 比 0.30 更好（更严格）

## 5. 放大利润

### 双方一致同意的

**不要直接放大 size，先做：**

1. **Walk-forward 验证** — 分段看参数稳定性
2. **交易成本压力测试** — bid/ask spread + 0.02、10-30% 成交失败、1-3 秒延迟
3. **只有通过压力测试的参数才值得实盘**

### 我们的方法

**直接放大 Size**
```
$10 → $50/笔 → +$600/213窗口
$10 → $100/笔 → +$1,200/213窗口
```
线性放大，不改变胜率。缺点：风险也线性放大。

**多窗口同时交易**
5 分钟窗口彼此独立，可同时持仓 2-3 个

**扩展到其他品种**
ETH、SOL、15 分钟窗口

### Codex 的方法

**分批止盈** — 0.70 出一半，0.85 出另一半
**动态仓位** — A 级信号全仓，B 级半仓，C 级不交易
**连亏降仓** — 连续亏损后自动缩小仓位
**组合策略** — FOLLOW + Momentum + Mean Reversion + Late-window
**多时段建模** — 亚/欧/美盘分别调参数
